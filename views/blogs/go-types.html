The type system is the most powerful feature for any programming language, types allows you to organize and store your data in any application.

Go is a statically typed language, which means compiler needs to know in advance that what is the type is for every value in the program. When a compiler knows the type information in advance, it helps to make sure that program running with values in safe way.

Go follows a minimalistic approach for its type system. Go provides several built-in types such as int, bool, float64. And also several composite types such as map, slices, channel which are made up of both built-in and user-defined types.

» Built-in types

Built in types are the one that provided by language itself. Below are the some useful built-in types in go.

// uint8 is the set of all unsigned 8-bit integers.
type uint8 uint8

// uint16 is the set of all unsigned 16-bit integers.
type uint16 uint16

// uint32 is the set of all unsigned 32-bit integers.
type uint32 uint32

// uint64 is the set of all unsigned 64-bit integers.
type uint64 uint64

// int8 is the set of all signed 8-bit integers.
type int8 int8

// int16 is the set of all signed 16-bit integers.
type int16 int16

// int32 is the set of all signed 32-bit integers.
type int32 int32

// int64 is the set of all signed 64-bit integers.
type int64 int64

// float32 is the set of all IEEE-754 32-bit floating-point numbers.
type float32 float32

// float64 is the set of all IEEE-754 64-bit floating-point numbers.
type float64 float64

// string is the set of all strings of 8-bit bytes, conventionally but not
// necessarily representing UTF-8-encoded text. A string may be empty, but
// not nil. Values of string type are immutable.
type string string

// int is a signed integer type that is at least 32 bits in size.
// It is a distinct type, however, and not an alias for, say, int32.
type int int

// uint is an unsigned integer type that is at least 32 bits in size.
// It is a distinct type, however, and not an alias for, say, uint32.
type uint uint

// byte is an alias for uint8 and is equivalent to uint8 in all ways. It is
// used, by convention, to distinguish byte values from 8-bit unsigned
// integer values.
type byte byte

» User-defined types

Go allows you to declare your own types. In go you can declare a user-defined type by using the struct keyword. In struct every field in user-defined type is declared from a known type, which can be a built-in type or another user-defined type.

type user struct {
    name string
    email string
    age int
    height float64
}

And you can declare a variable of above user type as :

// Initialization with zero values
var vikram user

// OR with initialization
vikram := user{
    name: "vikram",
    email: "xyz@gmail.com",
    age: 24,
    height: 5.11,
}

// OR in a single line without field names
vikram := user{ "vikram", "xyz@gmail.com", 24, 5.11 }

You can also declare a user-defined type for a built-in type as :

type Price float

In above code snippet even though Price is declared as float type but Go consider it as different type, which means you cannot assign float value directly. Below is an example.

package main

type Price float64

func main() {
    var price Price
    // Compiler error : cannot use float64(33.43) (type float64) as
    // type Price in assignment
    price = float64(33.43)
}
